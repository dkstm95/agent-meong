# Product Principles

## Product promise

agent-meong은 AI 에이전트에게 여러 작업을 맡긴 사람이 잠시 쉬면서도 작업이
계속되는지, 끝나가는지를 부담 없이 알아볼 수 있게 한다.

이 제품은 상세한 모니터링 도구가 아니다. 불멍처럼 한동안 바라볼 수 있는 차분한
움직임과, 다시 작업으로 돌아갈 시점을 알려주는 최소한의 상태 신호를 결합한다.

## Experience principles

### Calm, but informative

- 화면은 점과 올챙이처럼 단순한 유기적 형태를 중심으로 구성한다.
- 장식, 패널, 상하단 bar, 많은 숫자와 텍스트를 기본 화면에 추가하지 않는다.
- 사용자는 설명을 읽지 않아도 활동이 있는지와 줄어드는지를 대략 알아볼 수 있어야 한다.
- 기본 안내는 활동 중 움직임, 확인 필요 고리, turn 종료 파동의 세 신호만 먼저 보여주고,
  드문 결과 상태의 전체 아이콘 문법은 사용자가 `?` 도움말을 열었을 때 제공한다.
- 움직임과 오브젝트가 줄어들거나 사라지는 변화는 작업이 끝나간다는 신호가 된다.
- 활동 중 오브젝트는 popover 크기에 비해 정체돼 보이지 않는 속도로 이동한다. 가까운
  peer는 잠시 비켜가고, parent·child는 읽을 수 있는 거리대에서 맴돌 수 있지만 이
  장식적 상호작용을 새 observation 사실로 해석하지 않는다.
- 관찰된 parent 관계가 있을 때만 서브에이전트가 메인에서 태어나고 돌아가는
  lifecycle 움직임을 사용한다.
- 각 에이전트의 몸 색은 그 에이전트에서 실제로 관찰된 현재 상태를 나타내며, 상태
  전환마다 함께 바뀐다. 동시에 여러 에이전트가 있으면 각자의 상태 색을 독립적으로
  유지한다. 색을 구분하지 못해도 상태 형태와 VoiceOver로 같은 사실을 얻을 수 있어야 한다.
- 부모·자식 관계는 색에 고정하지 않고, 관찰된 관계에 따른 탄생·흡수와 가까운 관계
  움직임으로 표현한다.
- 메뉴 막대의 단일 아이콘은 여러 에이전트를 우선순위 한 색으로 집계하지 않는다.
  최근에 관찰된 에이전트 상태 전환을 색과 형태로 보여주고, 빠른 전환은 제한된 큐에서
  순서대로 표시한다. 활동 중인 에이전트가 하나 이상이면 움직임을 별도 신호로 유지하며,
  모든 동시 상태의 기준 화면은 Meong Space다.
- tool lifecycle은 짧은 일회성 움직임으로만 표현하고 지속 중인 tool 상태로 저장하거나
  추정하지 않는다. 겹쳐 보이는 점의 수와 출현 각도에 동시 실행 수나 tool 종류라는
  숨은 의미를 부여하지 않는다.

### Immediate and native

- 메뉴 막대 아이콘을 클릭하면 별도 창이 아니라 아이콘에 붙은 popover가 즉시 열린다.
- 외부 영역을 클릭하면 popover가 닫히는 일반적인 macOS 동작을 따른다.
- 메뉴 막대 아이콘은 발견 가능해야 하며, 실행 중인 작업이 있을 때만 절제된 활동을 보인다.
- 최상위 작업 하나의 종료 관찰은 다른 작업이 남아 있어도 짧은 변화와 확인 전까지
  해당 에이전트 가족에 남는 정적 신호로 알린다. 제한된 공간에서는 아직 확인하지 않은
  최근 가족만 보존하며 이를 전체 종료 횟수로 표현하지 않는다. 성공 여부는 source가
  명시적으로 제공할 때만 표현한다.
- 상태 확인을 위해 여러 화면을 탐색하거나 설정을 먼저 이해하게 만들지 않는다.

### Truthful state

- 오브젝트 수, lifecycle, 완료와 실패 상태는 실제 observation에서만 결정한다.
- 이동 경로, 미세한 속도 변화, 공간 배치 같은 장식적 움직임은 관찰 사실과 분리한다.
- 실제 활동이 없는데 바쁜 것처럼 보이게 만들지 않는다.
- 이벤트가 부족하거나 연결 상태를 확정할 수 없으면 성공으로 추정하지 않고 불확실성을 드러낸다.

### Onboarding is part of the product

- 설치와 연결 안내는 부가 문서가 아니라 핵심 사용자 경험이다.
- 첫 실행에서는 현재 지원하는 observation source와 적용되는 제품 surface를 즉시
  보여주고, 같은 연결을 중복 선택하게 하지 않는다.
- 첫 실제 observation 뒤에는 기본 상태 문법을 한 번만 짧게 안내하고, 이후에는
  Meong Space의 차분한 기본 화면을 유지한다.
- 안내 문구는 Codex나 다른 도구의 실제 UI와 lifecycle 용어를 정확히 사용한다.
- 앱이 직접 확인할 수 없는 trust, 권한, 연결 상태를 확인한 것처럼 표현하지 않는다.
- 앱이 설치, 검토용 CLI 실행, 상태 재확인을 맡고 사용자는 Codex가 강제하는 trust
  확인만 수행한다. `Trust all` 안내에는 다른 검토 대기 hook의 내용이 아닌 개수만
  사용한다.
- 개발용 프로젝트 로컬 연결과 일반 사용자용 설치 방식을 명확히 구분한다.
- 일반 checkout에는 자동으로 실행되는 project hook을 포함하지 않아 사용자 hook과
  중복 실행되거나 신뢰 항목이 늘어나지 않게 한다.

## Architecture principles

- macOS 메뉴 막대 앱을 첫 제품 범위로 유지한다.
- 내부 observation protocol은 AI 모델과 agent 도구에 종속되지 않아야 한다.
- Codex, Claude Code, 외부 CLI 같은 도구별 차이는 교체 가능한 adapter가 흡수한다.
- 앱은 prompt, response, 명령, 파일 경로, tool input/output을 수집하지 않는다.
- observation의 논리 ID는 원문 대신 고정 길이 digest만 허용하고, socket에서도 키와
  값 문법을 모두 검증한다.
- 식별자는 가능한 한 로컬에서 opaque 값으로 정규화하고 사용자 전용 IPC를 사용한다.
  설치 instance는 파일 경로에서 유도하지 않고 무작위 opaque 값으로 생성한다.

## Non-goals

- 여러 에이전트를 실행하거나 지휘하는 orchestration framework
- 로그, token, 명령, 파일 변경을 자세히 보여주는 운영 dashboard
- 별도 업무 관리 시스템이나 생산성 평가 도구
- 특정 모델 또는 provider에 최적화된 전용 client
- 실제 활동과 관계없는 지속적인 busy animation
- skill 실행, 승인·거절, 새 chat, reasoning 조절처럼 에이전트를 지휘하는 command
  center 기능

## Product checks

기능이나 디자인을 추가할 때 다음 질문을 확인한다.

1. 처음 보는 사용자가 색을 제외하고 3초 안에 활동 중, 확인 필요, turn 종료 신호를
   구분할 수 있는가?
2. 오브젝트가 줄어드는 것으로 복귀할 시점을 자연스럽게 파악할 수 있는가?
3. 1~3분 동안 바라봐도 시각적으로 피곤하거나 정보가 과하지 않은가?
4. 화면에 보이는 상태가 실제 observation보다 더 많은 사실을 암시하지 않는가?
5. 연결과 권한 안내가 현재 지원 도구의 실제 동작과 일치하는가?
6. 이 기능이 ambient awareness를 강화하는가, 아니면 dashboard 복잡도를 추가하는가?
7. Reduce Motion, Increase Contrast, VoiceOver에서도 같은 관찰 사실을 얻을 수 있는가?
8. 한 에이전트의 상태 변화가 다른 에이전트의 우선순위 상태에 가려지지 않는가?
